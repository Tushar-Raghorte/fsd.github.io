Development Standards

Coding Guidelines & Best Practices

During the Full Stack Development Training Programme, every participant is expected to follow industry-standard coding practices, maintain clean code, and use professional development workflows.

📁

Project Structure

Maintain a clean folder structure with separate modules, reusable components, and meaningful file names.

📝

Naming Conventions

Use descriptive names for variables, functions, classes, components, and APIs. Avoid abbreviations.

🧹

Clean Code

Write readable, maintainable code with proper formatting, indentation, and minimal duplication.

🔄

Version Control

Commit frequently with meaningful commit messages and push your work to GitHub regularly.

🛡️

Error Handling

Handle exceptions gracefully, validate inputs, and provide user-friendly error messages.

⚡

Performance

Optimize applications by minimizing unnecessary API calls, reducing bundle size, and writing efficient queries.

🔒

Security

Never hardcode passwords or API keys. Validate user input and follow secure authentication practices.

📚

Documentation

Maintain a README file, document APIs, and add comments only where they improve understanding.

🧪

Testing

Test every feature before submission and verify functionality across different browsers and devices.

🤝

Team Collaboration

Participate in code reviews, communicate effectively, and resolve merge conflicts professionally.

🚀

Continuous Learning

Read official documentation, practice daily, and keep yourself updated with the latest technologies.

🏆

Professionalism

Submit assignments on time, follow coding standards, and maintain high-quality project deliverables.

Code of Conduct

Training Programme Rules

To maintain a professional learning environment and ensure maximum benefit from the Full Stack Development Training Programme, every participant is expected to follow the guidelines below.

01

Training Schedule

Training begins every day at 10:45 AM. Students should be seated and ready before the session starts.

02

Punctuality

Students arriving late will not be permitted to join the session without prior approval from the Head of Department (HOD). Please arrive at least 10–15 minutes early.

03

Bring Your Laptop

Every student must bring their own laptop every day for hands-on practice, coding exercises, quizzes, and assignments.

04

Internet Connectivity

Students are advised to carry their own mobile hotspot or ensure a reliable internet connection for uninterrupted lab sessions.

05

Professional Behaviour

Video recording, gaming, social media browsing, and any unrelated activities are strictly prohibited during training. Disciplinary action may be taken if violations are observed.

06

Daily Quiz

A mandatory quiz consisting of 5 MCQ questions will be conducted every day.

07

Google Classroom

Google Classroom will be used to share study materials, assignments, announcements, and reference resources.

08

Assignments

Every assignment must be completed individually. Assignments will be reviewed by the Head of Department.

09

Code Quality

Follow coding standards, maintain proper project structure, and write clean, reusable, and documented code.

10

Attendance

Maintain at least 90% attendance to receive the programme completion certificate.

11

Practice Daily

Spend at least 1–2 hours every day practicing coding after the training session.

12

Respect Everyone

Maintain discipline, respect trainers and fellow students, and contribute to a positive learning environment.
